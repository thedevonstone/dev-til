## Async/Await error handling

Wrapping a single `await` in try/catch is fine for one-offs, but for a
chain of async calls it gets noisy fast. A cleaner pattern when you want
the error without crashing:

```js
async function safe(promise) {
  try {
    return [null, await promise];
  } catch (err) {
    return [err, null];
  }
}

const [err, data] = await safe(fetchUser(id));
if (err) { /* handle */ }
```

Inspired by Go's error handling style. Works well when you want to
handle the error locally without rethrowing.

## Array.at()

`Array.at(-1)` returns the last element without `arr[arr.length - 1]`.
Works on strings too.

```js
const last = [1, 2, 3].at(-1); // 3
const lastChar = 'hello'.at(-1); // 'o'
```

## Object.hasOwn()

Prefer `Object.hasOwn(obj, key)` over `obj.hasOwnProperty(key)`.
`hasOwnProperty` can be shadowed if someone sets it as a key on the object.
`Object.hasOwn` is static and always works correctly.

## Object.entries iteration pattern

Cleanest way to iterate over an object's key-value pairs.

```js
for (const [key, value] of Object.entries(config)) {
  console.log(key, value);
}

// Or transform into a new object
const doubled = Object.fromEntries(
  Object.entries(prices).map(([k, v]) => [k, v * 2])
);
```

## Promise.allSettled vs Promise.all

`Promise.all` rejects as soon as any promise rejects — all-or-nothing.
`Promise.allSettled` waits for every promise and returns all results including failures.

```js
const results = await Promise.allSettled([fetchA(), fetchB(), fetchC()]);
results.forEach(r => {
  if (r.status === 'fulfilled') use(r.value);
  else log(r.reason);
});
```

Use `allSettled` when you want partial results.

## Array spread vs Array.from

Both convert iterables to arrays, but `Array.from` takes an optional map function.

```js
// spread
const arr = [...nodeList];

// Array.from with map — one step instead of two
const values = Array.from(nodeList, node => node.textContent);
```

`Array.from` also works on array-like objects (e.g. `arguments`) that spread doesn't handle.

## Array.at() for negative indexing

`Array.at(-1)` returns the last element without needing `arr[arr.length - 1]`.
Works on strings too.

```js
const last = [1, 2, 3].at(-1);     // 3
const secondLast = 'hello'.at(-2); // 'l'
```
